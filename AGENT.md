# Karpathy Coding Guidelines

## Think before coding

- State assumptions before implementation.
- Surface ambiguity and trade-offs instead of hiding them.
- Prefer the simplest approach that satisfies the request.

## Keep it simple

- Write only the minimum code required.
- Avoid speculative features, abstractions, and configuration.
- Simplify solutions that are more complex than necessary.

## Make surgical changes

- Change only what the request requires.
- Preserve adjacent code and its existing style.
- Remove only unused code created by the change.

## Define and verify success

- Turn each task into a concrete success criterion.
- For multi-step work, define a check for every step.
- Verify the result before declaring the task complete.
